# Writing Plans

Translate an approved design into an execution plan that sub-agents can follow reliably.

## Plan Principles

- Break work into short, coherent tasks.
- Make verification explicit for every task.
- Bias toward DRY, YAGNI, and minimal changes.
- Include testing before code completion, not after.

## Deliverable

Write a plan artifact with:

- The goal and architecture summary.
- Ordered tasks and target executors.
- Verification steps for each task.
- Review notes and completion criteria.
