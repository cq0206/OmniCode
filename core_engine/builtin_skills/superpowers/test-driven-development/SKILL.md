# Test-Driven Development

Execution should favor a red-green-refactor rhythm whenever practical.

## Expected Loop

1. Define or update the failing check first.
2. Observe the failure or reproduce the gap.
3. Implement the smallest change that satisfies the check.
4. Re-run verification and confirm the behavior.
5. Refactor only after the checks are green.
